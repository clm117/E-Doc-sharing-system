# 数据库迁移指南：从Oracle到SQLite

## 概述
本指南提供了将电子文档共享系统的数据库从Oracle迁移到SQLite的详细步骤。

## 准备工作
1. **安装Python 3**：确保系统已安装Python 3.6+，并添加到环境变量中
2. **安装依赖**：
   ```bash
   pip install cx_Oracle  # 仅当需要从Oracle迁移数据时
   ```
3. **备份Oracle数据**：在迁移前，建议备份Oracle数据库中的数据

## 迁移步骤

### 1. 创建SQLite数据库和表
运行以下命令创建SQLite数据库和表结构：

```bash
python migrate_to_sqlite.py
```

### 2. 迁移数据
迁移脚本会自动从Oracle数据库读取数据并导入到SQLite中。

### 3. 验证迁移结果
迁移完成后，脚本会显示迁移的记录数，验证数据是否成功迁移。

## 系统配置

### 1. 数据库连接配置
系统会优先使用Oracle数据库，如果Oracle不可用，则自动切换到SQLite模式。

### 2. SQLite数据库文件
SQLite数据库文件位于：`docshare.db`

### 3. 环境变量
无需特殊环境变量配置，系统会自动检测并使用可用的数据库。

## 功能测试

### 测试步骤
1. 启动应用：
   ```bash
   python app.py
   ```
2. 访问首页：`http://localhost:5000`
3. 测试核心功能：
   - 文件上传/下载
   - 支付流程
   - 订单管理

### 预期结果
- 系统正常启动，无错误信息
- 数据库操作正常，能够读取和写入数据
- 所有功能模块正常运行

## 故障排除

### 常见问题
1. **Oracle连接失败**：
   - 检查Oracle数据库是否运行
   - 验证连接参数是否正确
   - 确保cx_Oracle模块已正确安装

2. **SQLite文件权限问题**：
   - 确保当前用户有读写`docshare.db`文件的权限

3. **数据迁移失败**：
   - 检查Oracle数据库中的表结构是否与迁移脚本兼容
   - 查看迁移日志，定位具体错误

## 注意事项

1. **数据类型转换**：
   - Oracle的DATE类型会转换为SQLite的TEXT类型（ISO格式字符串）
   - Oracle的NUMBER类型会转换为SQLite的INTEGER或REAL类型

2. **约束处理**：
   - SQLite支持基本的约束（主键、唯一约束、外键）
   - 某些Oracle特有的约束可能需要在应用代码中实现

3. **性能考虑**：
   - SQLite适合中小型应用，对于大型应用可能需要考虑其他数据库
   - 建议定期备份`docshare.db`文件

4. **兼容性**：
   - 系统同时支持Oracle和SQLite，可根据需要切换
   - 迁移后，应用代码会自动使用可用的数据库
