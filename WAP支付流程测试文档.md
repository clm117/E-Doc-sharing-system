# WAP支付流程测试文档

## 测试环境
- 后端服务：http://127.0.0.1:5000 或 http://192.168.100.174:5000
- 数据库：Oracle (system/oracle123)
- 支付宝：模拟环境（使用qr.alipay.com作为示例）

## 测试流程

### 1. 测试支付页面访问
**步骤：**
1. 在浏览器中访问：`http://127.0.0.1:5000/mobile_payment_simple?session_id=test123456&file_price_type=1`

**预期结果：**
- 页面正常显示
- 订单号自动生成（格式：ORD{年月日时分秒}{4位随机数}）
- 支付金额显示：¥3.00（从payment_config表读取）
- 创建时间显示当前时间

**检查点：**
- [ ] 订单号格式正确
- [ ] 金额正确显示
- [ ] 页面样式正常

---

### 2. 测试WAP支付接口调用
**步骤：**
1. 点击"立即支付"按钮
2. 打开浏览器开发者工具（F12）→ Network标签
3. 查看是否发起了POST请求到 `/alipay_wap_pay`

**预期结果：**
- 按钮状态变为"正在跳转支付..."
- 显示加载提示："正在跳转到支付宝支付页面，请稍候..."
- Network标签中显示POST请求到 `/alipay_wap_pay`
- 请求体包含：`session_id`, `file_id`, `file_price_type`

**检查点：**
- [ ] 按钮被禁用
- [ ] 加载提示显示
- [ ] POST请求发送到 `/alipay_wap_pay`
- [ ] 请求体包含正确的参数

**后端日志检查：**
```
收到支付宝WAP支付请求
从payment_config表获取金额: 3.00 (价格类型: 1)
使用传递的文件: test123456, 文件名: 学习资料
生成订单号: ORD202601181230451234, 金额: 3.00, 文件: 学习资料
```

---

### 3. 测试WAP支付接口返回
**步骤：**
1. 在开发者工具中查看 `/alipay_wap_pay` 的响应

**预期结果：**
- 返回状态码：200
- 返回JSON格式：
```json
{
    "success": true,
    "pay_url": "https://qr.alipay.com/fkx17836y4q9z9h2p6q7c6d?out_trade_no=ORD202601181230451234&total_amount=3.00",
    "out_trade_no": "ORD202601181230451234",
    "message": "支付初始化成功"
}
```

**检查点：**
- [ ] success字段为true
- [ ] pay_url字段包含支付宝URL
- [ ] out_trade_no字段包含订单号
- [ ] message字段包含成功消息

---

### 4. 测试跳转到支付宝页面
**步骤：**
1. 点击返回的pay_url链接（或等待自动跳转）

**预期结果：**
- 跳转到支付宝支付页面（模拟环境）
- 页面显示支付信息

**注意：**
- 当前使用的是模拟的支付宝URL（qr.alipay.com）
- 实际项目中应该跳转到真实的支付宝支付页面

### 5. 测试异步通知接口
**步骤：**
1. 使用curl或Postman模拟支付宝异步通知

**curl命令：**
```bash
curl -X POST http://127.0.0.1:5000/alipay_notify \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "trade_status=TRADE_SUCCESS&out_trade_no=ORD202601181230451234&trade_no=2026011200000000000123456&total_amount=3.00&gmt_payment=2026-01-18%2014%3A30%3A45&buyer_id=2088102146225135&buyer_logon_id=test%40alipay.com"
```

**预期结果：**
- 返回字符串：`success`
- 后端日志显示：
```
收到支付宝异步通知
收到通知参数: {...}
交易状态: TRADE_SUCCESS, 订单号: ORD202601181230451234
订单不存在，创建新记录
交易记录更新成功: ORD202601181230451234
```

**检查点：**
- [ ] 返回`success`字符串
- [ ] 后端日志显示收到通知
- [ ] 数据库中创建了新记录
- [ ] 记录的trade_status为TRADE_SUCCESS

## 注意事项

1. **当前实现说明**
   - 支付宝接口调用是模拟的，返回的是qr.alipay.com的URL
   - 实际项目中需要集成真实的支付宝SDK
   - 签名验证功能已预留，但当前未实现

2. **数据库操作**
   - 当前代码在/alipay_wap_pay中不写入交易记录
   - 交易记录在/alipay_notify中创建或更新
   - 确保幂等性，避免重复处理

3. **安全建议**
   - 实际项目中必须验证支付宝回调的签名
   - 使用HTTPS协议
   - 设置合理的超时时间
   - 实现防重放攻击机制

4. **性能优化**
   - 为常用查询字段添加索引
   - 使用数据库连接池
   - 实现缓存机制
