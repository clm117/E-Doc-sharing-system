# 后台管理系统使用说明

## 一、系统概述

后台管理系统提供以下功能：
1. **文件管理**：添加、修改、删除文件信息，生成加密文件压缩包
2. **支付记录查询**：查询和查看支付宝支付记录
3. **支付配置维护**：管理支付参数和价格配置


## 二、启动系统

### 步骤1：安装依赖
```bash
pip install flask cx_Oracle qrcode
```

### 步骤2：启动后台管理系统
```bash
python admin.py
```

### 步骤3：访问后台管理界面
打开浏览器访问：`http://localhost:5001/admin`


## 三、功能说明

### 1. 文件管理

#### 1.1 添加文件
- **路径**：`/admin/add_file`
- **功能**：添加新文件到数据库
- **字段**：
  - 文件名称（必填）
  - 文件路径（必填）
  - 作者（选填）
  - 标准化文件名（选填）
  - 搜索关键字（选填）
  - 标签（选填）
  - ISBN号（选填）
  - 价格类型（必填）：标准版（¥3.00）或高级版（¥5.00）
  - 文件密码（选填）

#### 1.2 文件列表
- **路径**：`/admin/file_list`
- **功能**：查看所有文件，支持搜索
- **操作**：
  - 编辑文件信息
  - 删除文件
  - 搜索文件

#### 1.3 生成压缩包
- **路径**：`/admin/generate_package`
- **功能**：生成加密文件压缩包
- **参数**：
  - 文件ID列表（必填）
  - 压缩包名称（选填）
  - 输出路径（选填，默认为encrypted_files文件夹）

#### 1.4 编辑文件
- **路径**：`/admin/edit_file?file_id=xxx`
- **功能**：修改文件信息
- **字段**：与添加文件相同

### 2. 支付记录查询

#### 2.1 支付记录列表
- **路径**：`/admin/payment_records`
- **功能**：查看最近100条支付记录
- **字段**：
  - 订单号
  - 文件名称
  - 支付金额
  - 支付状态
  - 创建时间
  - 支付时间
  - 文件密码

#### 2.2 支付状态
- **TRADE_SUCCESS**：支付成功
- **TRADE_FAILED**：支付失败
- **TRADE_PENDING**：待支付

### 3. 支付配置维护

#### 3.1 支付配置列表
- **路径**：`/admin/payment_config`
- **功能**：查看所有支付配置
- **字段**：
  - 配置ID
  - 价格类型
  - 金额
  - 支付URL
  - 描述
  - 状态（启用/禁用）

#### 3.2 新增配置
- **功能**：添加新的支付配置
- **参数**：
  - 价格类型（必填）
  - 金额（必填）
  - 支付URL（必填）
  - 描述（选填）
  - 状态（选填，默认启用）

#### 3.3 编辑配置
- **功能**：修改现有支付配置
- **操作**：点击"编辑"按钮，修改后保存

#### 3.4 删除配置
- **功能**：删除支付配置（待实现）


## 四、API接口

### 4.1 搜索文件
- **路径**：`/admin/api/search_files`
- **方法**：GET
- **参数**：
  - keyword：搜索关键字（选填）
  - file_type：文件类型（选填）
- **返回**：JSON格式的文件列表

### 4.2 删除文件
- **路径**：`/admin/api/delete_file`
- **方法**：POST
- **参数**：
  - file_id：文件ID（必填）
- **返回**：JSON格式的操作结果

## 六、注意事项

### 6.1 数据库连接
- 确保Oracle数据库已启动
- 确保数据库连接配置正确
- 确保数据库用户有足够的权限

### 6.2 文件路径
- 文件路径必须是绝对路径
- 确保文件存在且可访问
- 文件路径不包含中文或特殊字符

### 6.3 压缩包生成
- 确保有足够的磁盘空间
- 确保输出路径存在且可写
- 压缩包包含文件信息和文件内容

### 6.4 安全性
- 后台管理系统建议部署在内网
- 建议添加身份验证
- 建议使用HTTPS协议


## 七、常见问题

### 问题1：无法连接数据库
- **解决**：检查数据库连接配置，确保数据库已启动

### 问题2：文件添加失败
- **解决**：检查文件路径是否正确，确保文件存在

### 问题3：压缩包生成失败
- **解决**：检查磁盘空间，确保输出路径存在

### 问题4：支付记录查询失败
- **解决**：检查数据库表是否存在，确保表结构正确
